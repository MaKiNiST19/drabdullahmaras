Prozen – React Business & Consulting Template

Clean, modern, animated single-page application built with React 19, Vite and Tailored micro‑interactions.

React 19 Vite 7 SPA

1. Overview

Prozen React JS Template is a modern Single Page Application (SPA) designed for business, finance, corporate and consulting websites. It focuses on:

  • Three hero homepages (Main, Corporate, Insurance)
  • Fully React‑based routing – no jQuery, no inline scripts
  • Smooth scrolling powered by Lenis
  • Scroll animations handled by AOS (Animate On Scroll)
  • Reusable UI sections for services, projects, team, FAQ, pricing, blog, and more

2. Getting Started

2.1. Requirements

  • Node.js 18+ (recommended 20+)
  • npm 9+ or pnpm / yarn (optional)
  • A modern browser (Chrome, Edge, Firefox, Safari)

2.2. Installation

  1. Install dependencies

    cd prozen-react
npm install

  2. Run development server

    npm run dev

    By default Vite will start on http://localhost:5173.

  3. Build for production

    npm run build
npm run preview

2.3. Project Highlights

  • SPA navigation using react-router-dom@7
  • Context‑based Offcanvas menu for mobile navigation
  • Lenis for buttery‑smooth scroll and keyboard scroll
  • AOS for one‑time scroll‑in animations on every route change
  • Swiper for hero sliders and testimonials
  • React Hook Form for accessible contact forms

   │  └─ LenisContext.jsx      # Global Lenis instance + keyboard scroll integration
   ├─ hooks/
   │  ├─ useCountUp.js         # Counter-on-visible animation
   │  ├─ useInputForm.js       # React Hook Form integration
   │  └─ useStickyHeader.js    # Sticky header based on scroll threshold
   └─ data/
      ├─ homeData.js           # Home page content (hero, services, projects, posts)
      ├─ homeTwoData.js        # Home Two content
      ├─ HomeThreeData.js      # Home Three content
      ├─ aboutUsData.js        # About content
      ├─ faqData.js            # FAQ and testimonials content

      ├─ teamAndDetailsData.js
      ├─ pricingData.js
      └─ projectData.js

The goal is to keep all route logic in `App.jsx`, all layout in `pages/*`, and all reusable sections in `components/pages/*`. Content (titles, copy, images) lives in the `data/*` files so your buyers can update text without touching JSX.

6. Layout & Routing

6.1. Entry – main.jsx

The app is bootstrapped with createRoot and global styles are imported here: 

import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "bootstrap/dist/css/bootstrap.min.css";
import "./style/style.css";
import "@fortawesome/fontawesome-free/css/all.min.css";
import "./style/lenis.css";

import App from "./App.jsx";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <App />
  </StrictMode>
);

6.2. Global providers – App.jsx

App.jsx wraps the entire SPA with providers and defines all routes:

  • LenisProvider – smooth scroll & keyboard handling
  • BrowserRouter – client-side routing
  • OffcanvasProvider – global mobile menu state
  • AOSInit – initializes and refreshes AOS on route changes

Global UI that should be visible on every page – Offcanvas menu, ScrollToTop, BackToTop, custom Cursor – is rendered outside of <Routes>.

7. Pages & Sections

7.1. Homepages

  • Home (`/`) – Business / consulting default homepage with hero, services, about, stats, projects, process, team, testimonials, blog.
  • Home Two (`/hometwo`) – Corporate layout with alternate hero, stats, service carousel, and FAQ testimonials.
  • Home Three (`/homethree`) – Insurance‑focused layout with hero video card, services, process, pricing, testimonials and FAQ.

7.2. Inner pages

  • About Us – Mission, process, stats, team, testimonials.
  • Services & Service Details
  • Projects & Project Details
  • Team & Team Details
  • Pricing – Pricing tiers + FAQ accordion.
  • Blog / Blog List / Blog Details
  • FAQ – React accordion (no Bootstrap JS).
  • Contact – Google Maps area + contact form (React Hook Form).

Each route-level page in src/pages/* is a thin wrapper that composes sections from src/components/pages/* and uses data from src/data/*.

8. Animations & Effects

8.1. Scroll animations (AOS)

AOS is initialized in AOSInit.jsx and refreshed on every route change using useLocation. Components opt into animations using data-aos attributes: 

<div
  data-aos="fade-up"
  data-aos-delay="300"
>
  ...
</div>

8.2. Smooth scrolling (Lenis)

LenisContext.jsx creates a single Lenis instance and wires it to a requestAnimationFrame loop and keyboard events. The custom ScrollToTop component integrates with Lenis to keep scroll positions in sync on every route change.

8.3. Sliders (Swiper)

Swiper is used for hero sliders and testimonials. Each instance imports only the required modules (e.g. Navigation, Autoplay) to keep bundle size small.

8.4. Micro‑interactions

  • Offcanvas mobile menu with animated submenus
  • Search overlay with smooth slide‑down animation
  • Back‑to‑top button that fades in after scroll
  • Custom cursor following mouse with hover states
  • Count‑up stats using useCountUp + IntersectionObserver

9. Customization

9.1. Colors & typography

Theme colors, typography and spacing are controlled in src/style/style.css using CSS variables (e.g. --ht-theme-color, --ht-heading-color). Updating these variables will update the entire template.

9.2. Content

All text, counters, and image URLs live in src/data/*:

  • homeData.js – main homepage hero, services, projects, posts.
  • homeTwoData.js, HomeThreeData.js – alternate layouts.
  • aboutUsData.js, faqData.js, blogData.js, etc.

9.3. Adding a new section

  1. Create a new component under src/components/pages/<group>/ (e.g. services, home).
  2. Wire its content via a new or existing data file.
  3. Include the component in the proper page in src/pages/*.

10. Performance Notes

  • Built on Vite – instant dev server and optimized production bundles.
  • Only necessary Swiper modules are imported per slider.
  • Animations are lightweight (CSS + AOS), no heavy JS animation engines.
  • Lenis smooth scroll is created once and shared via React context.

For further optimization you can enable route‑level code splitting using React.lazy in App.jsx.

11. Frequently Asked Questions

11.1. Does this template use jQuery?

No. All interactivity is implemented in React and vanilla JS.

11.2. Can I use only one of the homepages?

Yes. Remove unused route imports from App.jsx and their corresponding navigation items from the header and offcanvas menu.

11.3. How do I change images?

Update the image paths in the data files (e.g. homeData.js, aboutUsData.js) and place your assets under the public images/ directory.

11.4. How do I change the primary color?

Edit the CSS variable --ht-theme-color inside src/style/style.css.
